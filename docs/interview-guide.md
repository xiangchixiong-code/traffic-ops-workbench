# 面试讲解稿

这份文档用于面试前快速复习，避免只会说“我做了一个 Spring Boot 项目”，但讲不清业务流程。

## 1. 项目一句话介绍

这是一个交通信息化工单与设备台账系统，模拟运营商政企/DICT 项目里常见的设备资产管理、巡检记录、故障工单处理、统计看板和 Excel 台账导入导出场景。

## 2. 为什么做这个项目

运营商、国企和政企信息化项目里，经常需要给交通、园区、政务等客户做后台系统。系统里通常会有设备台账、问题工单、巡检记录、统计报表、接口文档和数据导入导出。本项目就是围绕这些常见需求做一个小而完整的后端演示。

## 3. 可以重点讲的模块

- 设备台账：记录设备名称、资产编号、设备类型、状态、所属路段、安装位置和安装时间。
- 故障工单：针对设备创建故障工单，并支持工单状态流转。
- 巡检记录：记录运维人员对设备的巡检结果和备注。
- 统计看板：统计设备状态、工单状态、设备数量和路段数量。
- Excel 导入导出：设备台账可以导出为 Excel，也可以通过 Excel 批量导入或更新。
- 接口文档：使用 Knife4j/Swagger 展示和调试接口，方便前后端联调。

## 4. 面试官可能问什么

### 你负责了什么？

我主要负责后端接口设计与实现，包括设备台账、故障工单、巡检记录、统计看板和 Excel 导入导出等模块；同时整理 README、接口说明、启动方式和测试说明。

### 为什么选交通信息化？

因为运营商政企和国企信息化岗位经常会接触交通、政务、园区等行业客户。交通设备台账、巡检和故障工单比较容易抽象成后台系统，也适合用来练习业务建模、接口开发和问题流转。

### 这个项目和运营商岗位有什么关系？

它对应运营商政企支撑里常见的客户设备管理、工单流转、接口联调、数据台账整理和运维统计场景。虽然项目规模不大，但流程和岗位工作比较接近。

### Excel 导入导出怎么做的？

项目使用 Apache POI 处理 Excel 文件。导出时把设备台账数据写入工作簿；导入时读取 Excel 表头和行数据，根据资产编号判断是新增设备还是更新已有设备。

### 如果继续完善，你会加什么？

我会优先补充用户登录、角色权限、工单派发、附件上传、短信或邮件通知、设备告警接入和前端管理页面，让项目更接近真实交付系统。

## 5. 面试演示顺序

1. 打开 README，说明项目背景和模块图。
2. 启动项目，打开 Knife4j 页面。
3. 先调用 `GET /api/dashboard` 看统计数据。
4. 调用 `GET /api/devices` 查看设备台账。
5. 调用 `POST /api/tickets` 创建故障工单。
6. 调用 `PATCH /api/tickets/{id}/status` 更新工单状态。
7. 调用 `GET /api/devices/export` 导出 Excel。
8. 说明 MySQL profile 和自动化测试。
